将此规范文档分发给开发成员（或作为你自己的开发准则），将 `app.css` 引入 SvelteKit 项目，即可保证所有页面风格高度统一。

# 1. OpenBrain 前端开发规格说明书 (Design Guidelines)

## 1.1 设计哲学 (Design Philosophy)
*   **隐喻：** “未来的数字图书馆”。既要有象牙塔的静谧（纸张感、衬线体），又要有现代科技的高效（清晰的层级、极速的交互）。
*   **去噪：** 严禁使用复杂的渐变背景、霓虹光效或拟物化 AI 机器人图标。
*   **内容优先：** 数据库是核心。UI 只是容器，应当退后一步，让内容（标题、摘要）成为主角。

## 1.2 视觉原子 (Visual Atoms)

### A. 色彩体系 (Color Palette)
我们采用一套高雅、低饱和度的配色方案。

| 变量名 | 色值 | 语义/用途 |
| :--- | :--- | :--- |
| **Primary** | `#2C3E50` (深靛蓝) | **核心色**。用于标题、主按钮、强调边框。象征深度思考。 |
| **Accent** | `#E67E22` (陶土橙) | **点缀色**。用于链接悬停、高亮关键词、次要标签。象征活力。 |
| **Background**| `#F9FAFB` (纸张白) | **全局背景**。非纯白，带有微弱的灰调，保护视力。 |
| **Surface** | `#FFFFFF` (纯白) | **卡片/输入框背景**。用于承载内容。 |
| **Text Main** | `#1F2937` (墨灰) | 正文文本。 |
| **Text Sub** | `#6B7280` (中灰) | 辅助说明、元数据、日期。 |
| **Success** | `#10B981` (翠绿) | 状态指示（如：AI 生成完毕）。 |

### B. 字体排印 (Typography)
这是“人文气息”的关键来源，必须严格执行。

*   **标题 (Headings):** 使用 **衬线体 (Serif)**。
    *   *Font Stack:* `"Noto Serif SC", "Songti SC", serif`
    *   *应用:* `h1`, `h2`, `h3`, 品牌 Logo。
*   **正文/UI (Body/UI):** 使用 **无衬线体 (Sans-Serif)**。
    *   *Font Stack:* `system-ui, -apple-system, Roboto, sans-serif`
    *   *应用:* 导航、按钮、正文段落、表单输入、列表。

### C. 形状与质感 (Shape & Texture)
*   **圆角 (Radius):** 统一使用 `8px` (小组件) 或 `12px` (大卡片)。**严禁**使用完全圆形的“胶囊按钮”（Tags 除外）。
*   **阴影 (Shadow):** 仅在悬浮或层级分离时使用。追求“纸张浮起”的自然感，而非“发光”。
*   **毛玻璃 (Blur):** 仅用于顶部导航栏 (`backdrop-filter: blur(10px)`).

### 约束总结 Checklist
在提交代码前，请检查：
1.  **字体检查**：标题是否是衬线体？正文是否是无衬线体？
2.  **颜色检查**：是否使用了 `#2C3E50` 作为主色？是否有刺眼的亮色？
3.  **圆角检查**：是否统一使用了 `8px` 或 `12px`？
4.  **间距检查**：元素之间是否拥挤？（卡片内边距通常不少于 `1.5rem`）。
